# Security Policy

## Supported scope

This repository is public, but the deployed application may remain private.

If you find a security issue in:

- authentication
- authorization
- tenant isolation
- file import handling
- sensitive financial data exposure

please report it privately instead of opening a public issue with exploit details.

## Reporting

Until a dedicated disclosure channel exists, use a private contact path that you control and avoid posting proof-of-concept exploits in public.

Good reports usually include:

- affected route or page
- reproduction steps
- expected behavior
- actual behavior
- impact assessment

## Notes

- Do not include real personal or banking data in reports.
- Keep any testing limited to environments you own or are explicitly authorized to assess.
